# Creator Fees

Pump.fun rewards token creators with a share of trading fees generated by their token. This page covers how creator fees work, how to claim them using the SDK, and how to manage fee splits across multiple wallets.

---

## How Creator Fees Work

Every trade on a Pump.fun token generates a fee. A portion of that fee is allocated to the token's creator based on a dynamic, market-cap-tiered model.

### Fee Tiers

| Market Cap Range | Creator Fee Rate |
| ---------------- | ---------------- |
| < \$88,000       | 0.05%            |
| \$88,000–\$300,000 | 0.95%          |
| > \$300,000      | Scales down      |

Fees accrue automatically with every trade. Creators do not need to take any action until they are ready to claim.

### Pool Types

Creator fees accumulate in two pool types depending on where the token is trading:

- **pump** — The bonding curve pool (pre-migration).
- **meteora-dbc** — The Meteora Dynamic Bonding Curve pool (post-migration to Raydium/Meteora).

---

## Fee Ownership

By default, creator fees are assigned to the **wallet that deployed the token**. If you launch from your Phantom wallet, fees accrue to that same address.

### Fee Sharing

Creators can split fees across **up to 10 wallets** after launch. This is useful for teams, collaborators, or community-driven tokens.

- Fee percentages are assigned per wallet through the Pump.fun web or mobile UI.
- When any single recipient claims, distribution triggers automatically for all assigned wallets.
- Unclaimed fees remain available indefinitely — there is no expiration.

### Ownership Transfer

Token ownership can be fully transferred to a different wallet. The new owner inherits the creator fee stream. This enables clean handoffs for:

- Community Takeovers (CTOs)
- Team transitions
- Narrative-driven tokens where the "face" of the project changes

### CTO Fee Applications

Community Takeover administrators can apply to receive creator fees on tokens they did not originally deploy. This is managed through the Pump.fun UI and does not require SDK interaction.

## Claiming Fees with the SDK

The SDK provides a single tool for claiming accumulated creator fees: `pump_claim_fees`.

### Parameters

| Parameter       | Type     | Required | Description                                                                 |
| --------------- | -------- | -------- | --------------------------------------------------------------------------- |
| `userPublicKey` | `string` | Yes      | Creator's Base58 wallet public key (the wallet entitled to fees).           |
| `mint`          | `string` | No       | Token mint address. Omit when using the `pump` pool to claim all at once.   |
| `pool`          | `string` | No       | `"pump"` (default) or `"meteora-dbc"`. Determines which pool to claim from. |
| `priorityFee`   | `number` | No       | Priority fee in SOL. Default: `0.000001`.                                   |

### Usage

#### Claim all fees from the bonding curve pool

```
Tool: pump_claim_fees
{
  "userPublicKey": "YourBase58PublicKey"
}
```

The `pump` pool is the default. Omitting `mint` claims fees across all your tokens in a single transaction.

#### Claim fees for a specific token on Meteora

```
Tool: pump_claim_fees
{
  "userPublicKey": "YourBase58PublicKey",
  "mint": "TokenMintAddress",
  "pool": "meteora-dbc"
}
```

### Signing Flow

1. The SDK builds an unsigned transaction via the PumpPortal API.
2. A signing session URL is returned.
3. Open the URL in your browser, connect your wallet, and sign the transaction.
4. Fees are transferred to your wallet on confirmation.

---

## Key Things to Know

- **Fees accrue automatically.** No setup beyond launching the token is required.
- **No expiration.** Unclaimed fees stay available forever.
- **Batch claiming.** On the `pump` pool, omit `mint` to claim fees from all your tokens at once.
- **Two pools.** If your token has migrated, claim from both `pump` and `meteora-dbc` to collect everything.
- **Fee splits are off-chain.** Multi-wallet fee sharing is managed through the Pump.fun UI, not the SDK. The SDK claims for whichever wallet you provide.
