pumpsdk

A Model Context Protocol server that gives AI assistants 35 tools for launching tokens, trading, managing creator fees, real-time streaming, analytics, and autonomous scouting on Pump.fun.

Quick Start

Install and run with a single command — no cloning required:

npx pumpsdk

Environment Variables

Create a .env file or export these before running:

VariableRequiredDescription
SOLANA_RPC_URLRequiredSolana RPC endpoint (mainnet or devnet). Falls back to the public mainnet endpoint.
FAL_API_KEYOptionalfal.ai API key for Nano Banana 2 Pro image generation. Required for Scout image gen.
ANTHROPIC_API_KEYOptionalAnthropic API key for Claude Sonnet strategic decisions in agent mode.
NOUS_API_KEYOptionalNous Research API key for Hermes 4 fast routing in agent mode.
IMAGE_GEN_PROVIDEROptionalImage gen backend: fal (default) or replicate.

Claude Desktop

{

      "env": {

Transactions are unsigned

Every transaction returned by the server is unsigned. The user signs with their own wallet on a localhost page — the server never holds private keys.

Scout Launcher

The Scout Launcher is an AI-assisted token creation flow that handles trend scanning, image generation, metadata upload, and multi-phase signing in a single interactive page.

How It Works

When the scout generates a launch package, it creates a session on the localhost signing server (port 3142) and opens a dedicated scout page where the user reviews everything before committing.

Scout Session Flow

Preview
→
Connect Wallet
→
Set Dev Buy
→
Sign Launch Tx
→
Sign Fee Config
→
Done

Preview Phase

The scout page displays the AI-generated token — name, symbol, description, and logo image. The user can edit any field, regenerate the image with a custom prompt, or approve as-is.

Image Generation

Token logos are generated via fal.ai Nano Banana 2 Pro, a fast Gemini-based image model. The server constructs an optimized prompt from the token name, symbol, and description. Users can regenerate with a custom prompt from the scout page. Requires the FAL_API_KEY environment variable.

Dev Buy Slider

A SOL slider lets the user set their initial buy amount (the "dev buy") before signing. This determines how many tokens the creator receives at launch, separate from the token creation itself.

Multi-Phase Signing

On approval, the page walks through two sequential transactions:

  1. Launch transaction — Creates the token on-chain via the Pump.fun program. A mint keypair is pre-generated server-side so both phases reference the same mint address.
  2. Fee config transaction — Sets up creator fee sharing (wallet splits, basis points). This must come after launch because the fee program requires an initialized mint account.

Security

The signing server binds to 127.0.0.1 only (not 0.0.0.0). Every POST endpoint validates the Origin header against http://localhost:3142 and checks a 64-character hex CSRF token generated per session. Sessions expire after 10 minutes.

Creator Fees

Pump.fun rewards token creators with a share of trading fees. Fees accrue automatically with every trade — creators only need to act when they want to claim.

Fee Tiers

Market Cap RangeCreator Fee Rate
< $88,0000.05%
$88,000 – $300,0000.95%
> $300,000Scales down

Pool Types

Fees accumulate in two pool types depending on the token's trading phase:

  • pump — The bonding curve pool (pre-migration).
  • meteora-dbc — The Meteora Dynamic Bonding Curve pool (post-migration to Raydium/Meteora).

If your token has graduated, claim from both pools to collect everything.

Fee Ownership and Sharing

By default, fees accrue to the deployer wallet. Creators can split fees across up to 10 wallets after launch via the Pump.fun UI. When any single recipient claims, distribution triggers automatically for all assigned wallets. Unclaimed fees never expire.

Claiming with MCP Tools

Claim all fees from the bonding curve pool

Tool: pump_claim_fees
{
  "userPublicKey": "YourBase58PublicKey"
}

Omitting mint on the pump pool claims fees across all your tokens in one transaction.

Claim from a specific token on Meteora

Tool: pump_claim_fees
{
  "userPublicKey": "YourBase58PublicKey",
  "mint": "TokenMintAddress",
  "pool": "meteora-dbc"
}

Signing Flow

  1. The server builds an unsigned transaction via the Pump.fun API.
  2. A signing session URL is returned.
  3. Open the URL, connect your wallet, and sign the transaction.
  4. Fees transfer to your wallet on confirmation.

Ownership Transfer and CTOs

Token ownership can be fully transferred to a different wallet — the new owner inherits the creator fee stream. Community Takeover administrators can apply to receive creator fees on tokens they did not originally deploy through the Pump.fun UI.

Agent System

The agent layer adds autonomous operation capabilities through a dual-model orchestrator that routes tasks to the best model for the job.

Dual-Model Orchestrator

ModelProviderRoleEnv Key
Hermes 4 Nous Research Fast routine operations — claims, balance checks, monitoring, price quotes, holdings lookups NOUS_API_KEY
Claude Sonnet Anthropic Strategic decisions — analysis, evaluation, recommendations, optimization, scouting, graduation timing ANTHROPIC_API_KEY

The orchestrator uses pattern matching on the task description to route automatically. Routine patterns (claim, balance, check, list, monitor) go to Hermes for speed. Strategic patterns (evaluate, analyze, recommend, optimize, scout) go to Sonnet for depth.

Autonomous Strategies

Six built-in strategies can run as background loops:

auto-claim

Periodically checks for accrued creator fees and claims them when they exceed a configurable SOL threshold.

fee-optimizer

Analyzes fee configurations across tokens and recommends optimal split adjustments based on trading volume.

graduation-watch

Monitors bonding curve progress for tokens in your portfolio and alerts when they approach the ~85 SOL graduation threshold.

launch-monitor

Watches for new token launches matching keyword filters and surfaces promising opportunities.

sniper

Monitors new mints and builds trade transactions for tokens matching keyword criteria with configurable max SOL and slippage bounds.

scout

Periodically scans trending topics and generates launch packages ready for one-click deployment.

Session Store

Signing sessions are persisted to .sessions/sessions.json using a file-backed store with a process-level async mutex to prevent TOCTOU race conditions. Sessions auto-expire after 10 minutes (SESSION_TTL_MS = 600,000). Expired entries are pruned on every read.

CSRF Protection

Every session gets a 64-character hex CSRF token (32 random bytes). POST endpoints validate both the Origin header (must be http://localhost:3142 or absent for same-origin) and the CSRF token in the request body.

Transport Layer

TransportUse CaseDetails
stdio Claude Desktop, local CLI Default. Uses StdioServerTransport from the MCP SDK.
HTTP Remote connections, monitoring Express-based streamable HTTP on port 3000. Includes per-IP rate limiting (1000 req/hr).

WebSocket Client

The PumpPortal WebSocket client (client/pump-ws.ts) connects to wss://pumpportal.fun/api/data with exponential backoff reconnect (1s base, 30s max). Events are buffered in a 100-entry ring buffer so recent history is always available even between reconnects.

API Reference

The localhost signing server runs on port 3142 and exposes REST endpoints for three session types. All endpoints bind to 127.0.0.1 only.

Signing Sessions

For pre-built transactions (trades, claims, generic signing).

MethodEndpointDescription
GET/sign/:sessionIdServe the signing page HTML.
GET/api/sign/:sessionIdFetch session data: transactions, description, meta, RPC URL, CSRF token.
POST/api/sign/:sessionId/completeMark session complete with signature array. Validates origin and CSRF.

Launch Sessions

Multi-phase token launch: connect wallet, sign launch tx, sign fee config tx.

MethodEndpointDescription
GET/launch/:sessionIdServe the launch page HTML.
GET/api/launch/:sessionIdFetch session state: phase, description, meta, CSRF token.
POST/api/launch/:sessionId/connectPhase 1: Wallet connects. Builds launch tx for the connected wallet.
POST/api/launch/:sessionId/launch-signedPhase 2: Launch tx signed. Builds fee config txs using the now-live mint.
POST/api/launch/:sessionId/completePhase 3: Fee config signed. Session marked complete.

Scout Sessions

AI-generated token with image preview, editing, and multi-phase signing.

MethodEndpointDescription
GET/scout/:sessionIdServe the scout page HTML.
GET/api/scout/:sessionIdFetch scout session: token details, image URL, phase, CSRF token, mint public key.
POST/api/scout/:sessionId/updateEdit token name, symbol, or description. Optionally regenerate the image.
POST/api/scout/:sessionId/regenerateRegenerate the token image with the current prompt.
POST/api/scout/:sessionId/approveApprove: uploads to IPFS, builds launch tx. Accepts wallet and initialBuySol.
POST/api/scout/:sessionId/launch-signedLaunch tx signed. Builds fee config txs.
POST/api/scout/:sessionId/completeFee config signed. Session marked complete.
Session Expiry

All sessions expire after 10 minutes. Expired sessions are pruned automatically on the next API call.
